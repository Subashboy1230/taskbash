taskbash · architecture

Personal chief-of-staff agent. Reads Gmail, Granola, Linear, and Calendar every morning, extracts action items via Claude, deduplicates against what you have already cleared, drafts replies in your voice, and serves the result through a Next.js app on Vercel backed by Supabase Postgres and an Inngest cron.

External / third-party App code · extractors, orchestrator, actions Persistence + read surfaces Slop feedback loop Sidecar (cron, LLM call, trace stream)

Tech stack

VercelHosting
Next.js 15 App Router. Auto-deploys from main, preview URLs per dev branch. Custom domain taskbash.app via GoDaddy DNS.
SupabasePostgres + Auth
Postgres for all data (items, llm_calls, item_feedback, runs, user_functions). Auth via Google SSO with RLS on every table. Service-role client for server actions.
InngestCron + durable steps
Morning-digest cron at 8 AM local. Each step.run block is durable so a partial failure retries from the last checkpoint instead of replaying the whole pipeline.
NangoOAuth proxy
Handles Gmail + Google Calendar OAuth and proxies every API call. Stores per-user connection_id, refreshes tokens, never exposes the access token to our code.
AnthropicLLM
Claude Sonnet for brief synthesis (the moat) and reply drafting. Claude Haiku for extraction, classification, follow-up decisions. Every call wrapped by tracedMessage.
LangfuseObservability
Fire-and-forget LLM trace stream. Powers per-prompt-version slop_rate on /observability and gives ad-hoc inspection of any call by prompt_id.
GranolaSource
Meeting notes API, direct key. Extractor pulls the last 7 days of notes and feeds them to Claude for action-item synthesis.
LinearSource
GraphQL with a Personal API key. Filtered to issues where the user is mentioned in recent comments. No LLM call — pure heuristic.
